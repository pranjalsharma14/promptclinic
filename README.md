# PromptClinic  
**AI Prompt Diagnosis & Optimization Framework**

PromptClinic is a structured **Prompt Engineering Analysis Framework** designed to diagnose, evaluate, and optimize prompts for Large Language Models (LLMs) such as ChatGPT, Claude, and Gemini.

The project demonstrates how poorly structured prompts lead to weak AI outputs and how systematic prompt reconstruction dramatically improves results.

This repository showcases **advanced prompt engineering, output evaluation, and prompt optimization workflows** used in modern AI applications.

---

# Why PromptClinic?

Most AI users believe that poor AI results are caused by weak models.

In reality, the problem is usually **weak prompts**.

Common prompt issues include:

- Lack of clear objectives  
- Missing context  
- Undefined audience  
- No tone or style instruction  
- Missing constraints  
- No output format guidance  

Because of this, AI models generate **generic, vague, or misaligned outputs**.

PromptClinic solves this problem by introducing a **diagnostic framework for prompt engineering.**

# PromptClinic Framework

PromptClinic follows a structured prompt optimization cycle:

User Prompt  
↓  
Prompt Diagnosis  
↓  
Weakness Detection  
↓  
Prompt Repair  
↓  
Output Comparison  
↓  
Prompt Health Score

Each stage analyzes and improves the prompt to maximize output quality.

---

# Prompt Diagnosis Framework

The PromptClinic framework evaluates prompts across several dimensions:

### 1. Objective Clarity
Does the prompt clearly define what the AI must accomplish?

### 2. Context Completeness
Does the AI have enough background information?

### 3. Audience Definition
Is the intended audience specified?

### 4. Tone and Style Control
Does the prompt define tone, format, or writing style?

### 5. Constraints and Requirements
Does the prompt define boundaries, limits, or formatting rules?

### 6. Output Structure
Does the prompt guide how the response should be structured?

# Prompt Health Score

PromptClinic introduces a **Prompt Health Score** to evaluate prompt quality.

Each prompt is scored across five parameters:

| Category | Score Range |
|---------|------------|
| Objective Clarity | 0 – 10 |
| Context Quality | 0 – 10 |
| Instruction Precision | 0 – 10 |
| Output Structure | 0 – 10 |
| Constraints | 0 – 10 |

**Total Prompt Score = 50**

Interpretation:

- **45 – 50 → Excellent Prompt**  
- **35 – 44 → Good Prompt**  
- **25 – 34 → Weak Prompt**  
- **Below 25 → Poor Prompt**
